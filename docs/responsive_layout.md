# 全站屏幕适配说明

## 适配范围

第一轮全站屏幕适配只调整前端布局和样式，不改变业务流程、数据库结构和后端接口。

覆盖范围包括：全局容器、首页导航、账户/后台侧边栏、表格、表单按钮、商品列表、商品详情、个人中心和后台管理页。

## 主要断点

- `1200px` 以下：压缩侧边栏和页面间距，商品卡片列宽变小，详情页图片和信息区间距收紧。
- `900px` 以下：账户区和后台区从左侧栏布局切换为顶部横向导航，表格启用横向滚动。
- `720px` 以下：首页导航和搜索框换行，表单行、验证行、详情信息、订单信息改为单列。
- `520px` 以下：商品列表单列展示，表单按钮增强可点击尺寸，后台筛选和表格保持横向滚动。

## 验证页面

第一轮验证重点页面：

- 首页 `/`
- 商品详情 `/product/1`
- 登录 `/login`
- 个人中心 `/me`
- 钱包充值 `/wallet/recharge`
- 我买到的 `/me/bought`
- 账号设置 `/me/profile`
- 发布闲置 `/publish`
- 后台用户管理 `/admin/users`
- 后台订单管理 `/admin/orders`

验证宽度：

- 桌面宽屏：`1365px`
- 中屏：`900px`
- 小屏：`390px`

## 预期效果

- 页面主体不出现整体横向溢出。
- 表格在中小屏通过自身横向滚动查看，不挤压整个页面。
- 侧边栏在中小屏改为顶部横向导航，不再挤压主体内容。
- 表单和主要按钮在小屏下更容易点击。
- 商品列表在小屏下变为单列。
