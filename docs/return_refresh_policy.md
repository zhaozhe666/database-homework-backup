# 返回刷新规则

本项目不做全局返回强制刷新，只对数据容易变化的业务列表使用一次性刷新参数。

## 会刷新

- 商品详情返回市场：返回 `index?refresh=1`。
- 订单详情返回买到/卖出订单列表：返回对应订单列表并携带 `refresh=1`。
- 从提醒中心进入订单详情后返回提醒中心：保留提醒分类并携带 `refresh=1`。
- 消息、提醒操作完成后返回对应列表：携带 `refresh=1`。
- 发布、编辑、上下架、恢复、删除商品成功后返回我的发布：携带 `refresh=1`。

## 不强制刷新

- 个人资料、手机号、支付密码等设置页返回。
- 登录、注册、错误页返回。
- 后台普通管理页返回。

## 实现方式

列表页引入 `_refresh_once.html`。当 URL 中存在 `refresh=1` 时，页面完成服务端渲染后清理该参数，避免地址栏长期保留刷新标记。
