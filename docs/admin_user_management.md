# 管理员用户管理说明

## 停用用户

管理员在用户管理页点击“停用”后，该用户不能继续登录或访问需要登录的页面。用户再次登录时会看到提示：“账号已被管理员停用，请联系管理员处理。”

停用不会删除用户、订单、钱包余额、商品、评价和后台日志，适合保留历史业务记录的账号管控场景。

## 筛选用户

用户管理页支持按状态筛选“全部 / 启用 / 停用”，也可以继续通过搜索框搜索昵称、账号、手机号、管理员、启用或停用。

## 页面操作区

用户管理表格的信息较多，页面会保留横向滚动，并把“操作”列前移到“状态”后面，避免启用/停用按钮被权限操作区域挤出可视范围。

停用按钮使用独立的危险操作样式，文案为“停用账号”，提交前会弹出确认提示；恢复账号时显示“恢复启用”。交易统计按“发布 / 购买 / 卖出”三项独立展示，避免数字和说明文字错位。
