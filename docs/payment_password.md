# 支付密码与手机验证码

支付密码用于余额支付二次确认。

- 所有已有用户和初始化演示用户的默认支付密码为 `111111`。
- 支付密码按哈希值存入 `users.payment_password_hash`，不保存明文。
- 注册时必须填写手机号，并点击“发送验证码”后填写手机验证码；当前为测试流程，统一测试验证码为 `000000`。
- 个人中心的“修改手机号”为独立页面，需要填写验证码并分步验证；当前为测试流程，统一测试验证码为 `000000`。
- 个人中心的“修改支付密码”为独立页面，需要先填写原支付密码，再填写手机验证码后点击“确认修改”；当前为测试流程，统一测试验证码为 `000000`。
- 订单余额支付时必须输入 6 位数字支付密码。
