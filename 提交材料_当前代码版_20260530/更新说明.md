# 更新说明

本目录用于保存“当前代码版”的提交材料。旧版材料不直接覆盖，方便对照和回退。

## 目录说明

- `00_旧版重要参考_勿改`：保留早期旧版报告、PPT、ER 图、说明书和旧版 SQL，仅作参考。
- `01_第二版_已更新`：保存当前代码版报告、PPT、说明书和新版 ER 图。
- `02_当前运行数据库结构`：保存当前网站实际使用的数据库结构文件，以及当前功能详细说明 txt。
- `tools`：材料生成脚本或辅助工具。

## 当前代码版口径

- 数据库名：`secondhand`
- 当前核心表共 12 张：
  `users`、`admins`、`categories`、`products`、`product_images`、`favorites`、`orders`、`payments`、`app_settings`、`reviews`、`messages`、`notifications`
- 管理员采用独立表设计：
  `admins.user_id` 关联 `users.id`，账号密码仍统一保存在 `users` 表。
- 商品多图采用 `product_images` 表正式存储，`products.image_url` 仅作为兼容缓存。
- 本地上传目录：`代码/static/uploads/`
- 演示商品图片目录：`代码/static/product_images/`
- 所有会修改数据的 POST 表单都增加 CSRF token 校验，覆盖支付、退款、商品管理、用户管理等关键操作。
- 上传图片保存为系统生成的新文件名，中文原文件名只用于判断扩展名，避免 `图片.jpg` 这类合法文件被误判为格式不支持。
- 过期待支付订单清理不再每个请求都写库，改为只在首页、商品详情、订单详情、后台订单和个人订单等需要刷新状态的页面触发；支付接口仍保留超时兜底。
- 代码目录中的旧版 SQL / 临时 SQL 已清理，当前唯一准结构文件是 `代码/database/schema.sql`。

## 已完成的重要功能

- 商品发布、编辑、删除、下架、重新上架。
- 本地多图上传，一个商品最多 4 张图。
- 商品收藏与“我的收藏”。
- 站内消息，按联系人形成类似 QQ 的对话框，可回复、删除对话。
- 顶部提醒中心，显示未读消息、未读事件、卖家待发货订单和待处理退款数量；任务型提醒会避免重复计数，处理后自动清除旧提醒。
- 10 分钟待支付倒计时，超时自动取消订单并释放商品。
- 已付款订单拆分为 `paid` 待发货和 `shipped` 待收货；卖家可点击“已发货”，买家只能在已发货后确认收货。
- 已付款未收货订单可申请退款；买家可撤回退款，卖家可同意或拒绝退款，退款申请中不能直接确认收货，避免退款和收货并发导致资金错账。
- 卖家同意退款时会追加 `payments.status='refunded'` 流水，便于订单对账。
- 订单完成后买卖双方可互评。
- 个人中心可修改资料和密码。
- 管理员后台：
  商品管理、订单管理、用户管理、管理员注册开关、管理员下架商品并填写原因；商品、订单、用户列表均支持搜索。
- 关键安全与边界修复：
  CSRF 防护、退款状态闭环、中文文件名图片上传兼容。
- 维护性优化：
  过期订单清理减少不必要写库；删除 `schema_mysql56.sql`、`schema_sqlite.sql`、`校园二手交易平台数据库设计.sql`、`_dup.sql`，避免误改旧结构。
