# 校园二手交易平台（数据库课程设计）

基于 **Flask + MySQL 5.6** 的二手交易系统，完整覆盖：
注册登录 → 发布/浏览商品 → 下单（锁定商品）→ 余额支付（平台托管）→ 确认收货（打款卖家）→ 交易完成。

## 一、技术栈
- 后端：Python + Flask
- 数据库：MySQL 5.6（InnoDB，支持事务与外键）
- 数据库驱动：PyMySQL
- 前端：Jinja2 模板 + 原生 CSS

## 三、安装与运行（详细启动步骤）

> 全程在 **Windows + CMD（或 PowerShell）** 下操作。第一次运行请按 0→5 完整走一遍；
> 以后再启动只需做 **第 4 步**（数据库不用重复初始化）。

### 第 0 步：启动 MySQL 服务（必须先开，否则连不上数据库）
1. 按 `Win + R`，输入 `services.msc`，回车打开「服务」窗口。
2. 在列表里找到 MySQL 服务（名字通常是 `MySQL` 或 `MySQL56`）。
3. 右键 → 「启动」。如果「状态」一栏已显示「正在运行」，说明已开启，跳过即可。
4. （可选）也可用管理员身份的 CMD 启动：
   ```cmd
   net start mysql      :: 服务名以 services.msc 里看到的为准
   ```

### 第 1 步：打开命令行并进入代码目录
> 关键：一定要先 `cd` 到 `代码` 文件夹，否则会因为找不到 `app.py` 而报错。
```cmd
cd /d "C:\Users\赵喆\Desktop\数据库作业\代码"
```
> 小技巧：在文件资源管理器里打开 `代码` 文件夹，点击地址栏输入 `cmd` 回车，即可直接在该目录打开命令行。

### 第 2 步：安装依赖（仅首次需要）
```cmd
pip install -r requirements.txt
```
> 安装的是 Flask、PyMySQL、Werkzeug。已装过可跳过。

### 第 3 步：初始化数据库（仅首次，或想重置数据时）
```cmd

```
> 作用：自动建库 `secondhand`、建好所有表、写入 3 个演示账号和 6 件商品。
> ⚠️ 重复执行会**清空并重建**数据库（演示数据恢复原样），平时启动**不要**再跑这一步。
> 成功后会看到：`[OK] 数据库与数据表创建完成`、`[OK] 演示用户与商品写入完成`。

### 第 4 步：启动应用（每次使用都执行这一步）
```cmd
python app.py
```
> 看到下面这行就表示启动成功，**保持这个命令行窗口开着别关**：
> ```
>  * Running on http://127.0.0.1:5000
> ```

### 第 5 步：在浏览器访问
打开浏览器，地址栏输入：<http://127.0.0.1:5000>
然后用下方「演示账号」登录即可（如 alice / 123456）。

### 关闭应用
回到运行 `python app.py` 的命令行窗口，按 `Ctrl + C` 即可停止服务。

### 常见报错与解决
| 现象 | 原因 | 解决 |
|------|------|------|
| `Can't connect to MySQL server` / 连接超时 | MySQL 服务没启动 | 回到第 0 步在 services.msc 里启动 MySQL |
| `Access denied for user 'root'` | 密码不对 | 检查 `config.py` 里的密码是否为 `414290` |
| `Unknown database 'secondhand'` | 还没初始化数据库 | 先执行第 3 步 `python init_db.py` |
| `can't open file 'app.py'` | 没进入代码目录 | 先执行第 1 步的 `cd /d ...` |
| 浏览器打不开/拒绝连接 | 应用没在运行 | 确认第 4 步窗口仍开着且显示 Running |
| `Address already in use`（5000 被占用） | 已有一个实例在跑 | 关掉旧窗口，或在任务管理器结束多余的 python 进程 |

## 四、数据库设计
| 表 | 说明 |
|----|------|
| users | 用户（账号、加密密码、昵称、手机、钱包余额） |
| categories | 商品分类 |
| products | 商品（状态：在售/交易中/已售出/已下架） |
| orders | 订单（状态：待支付/待收货/已完成/已取消） |
| payments | 支付流水（余额支付） |

### 状态机
- 商品：`on_sale`(在售) → `locked`(交易中) → `sold`(已售出)；`on_sale` → `removed`(下架)
- 订单：`created`(待支付) → `paid`(待收货) → `completed`(已完成)；`created` → `cancelled`(已取消)

### 资金流（贴近现实的托管模式）
1. 买家下单：商品 `on_sale → locked`，生成 `created` 订单。
2. 买家支付：扣减买家余额（平台托管），订单 `created → paid`，记录支付流水。
3. 买家确认收货：托管资金打款给卖家余额，商品 `locked → sold`，订单 `paid → completed`。
4. 取消（仅待支付）：商品 `locked → on_sale`，订单 `created → cancelled`。

所有下单/支付/收货均在数据库事务中完成，并对商品、用户余额加行锁（`SELECT ... FOR UPDATE`），
避免并发下单/重复支付导致的数据不一致。

   ├─ app.py            # Flask 路由与业务逻辑（状态机、事务）
   ├─ db.py             # 数据库连接与查询/事务封装
   ├─ config.py         # 数据库与密钥配置
   ├─ init_db.py        # 初始化脚本（建库 + 演示数据）
   ├─ requirements.txt  # 依赖
   ├─ database/
   │  └─ schema.sql     # 建库建表 DDL + 基础分类
   ├─ templates/        # 页面模板
   └─ static/
      └─ style.css      # 样式
```
