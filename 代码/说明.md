# 校园二手交易平台（数据库课程设计）

本项目是基于 **Flask + MySQL 5.6** 的校园二手交易演示系统。系统围绕“发布商品、下单支付、发货收货、退款申诉、评价反馈、后台治理”形成完整业务闭环，适合作为数据库课程设计和答辩演示项目。

## 一、技术栈

- 后端：Python + Flask
- 数据库：MySQL 5.6，InnoDB，支持事务和外键
- 数据库驱动：PyMySQL
- 前端：Jinja2 模板 + 原生 CSS
- 图片：本地上传到 `static/uploads/`，演示商品图片放在 `static/product_images/`
- 安全：所有会修改数据的 POST 表单均带 CSRF token 校验
- 数据库结构：以 `database/schema.sql` 为准

## 二、运行环境

- Python：建议 3.8 及以上
- MySQL：5.6
- 默认数据库：`secondhand`
- 默认访问地址：`http://127.0.0.1:5000`
- 数据库连接配置：`config.py`

如本机数据库账号、密码、端口不同，请先修改 `config.py` 中的 `DB_CONFIG`。

## 三、启动方式

### 方式一：一键启动

进入 `代码` 目录，双击：

```text
一键启动.bat
```

脚本会检查 Python、依赖包和数据库连接；数据库可连接后，会启动 Flask 网站并自动打开浏览器。

脚本优先使用项目目录下的 `.venv` 虚拟环境；如果不存在，会自动创建并按 `requirements.txt` 安装依赖，避免使用系统全局旧版本依赖。

注意：一键启动不会自动执行 `init_db.py`，避免误清空演示数据库。

### 方式二：手动启动

1. 启动 MySQL 服务。

2. 进入项目代码目录：

```cmd
cd /d "C:\Users\赵喆\Desktop\数据库作业\代码"
```

3. 安装依赖：

```cmd
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

4. 首次运行或需要重置演示数据时初始化数据库：

```cmd
.venv\Scripts\python.exe init_db.py
```

注意：`init_db.py` 会清空并重建 `secondhand` 数据库，请不要在已有演示数据需要保留时执行。

5. 启动网站：

```cmd
.venv\Scripts\python.exe app.py
```

浏览器访问：

```text
http://127.0.0.1:5000
```

## 五、核心业务流程

### 1. 商品交易流程

1. 卖家发布商品，商品状态为 `on_sale`。
2. 买家下单后，商品锁定为 `locked`，订单状态为 `created`。
3. 买家需在 10 分钟内支付；未设置支付密码时需先完成设置，已设置后余额支付需要输入 6 位数字支付密码。
4. 支付成功后订单变为 `paid`，商品仍处于锁定交易中。
5. 卖家点击“已发货”，订单变为 `shipped`。
6. 买家确认收货后，订单变为 `completed`，商品变为 `sold`，金额打给卖家。

### 2. 超时取消

- 待支付订单超过 10 分钟未支付会自动取消。
- 订单状态变为 `cancelled`。
- 商品从 `locked` 恢复为 `on_sale`。
- 订单详情页会显示支付倒计时。

### 3. 退款流程

- 买家在已支付、未完成订单中可以申请退款。
- 订单状态变为 `refund_requested`。
- 卖家可以同意退款，也可以拒绝退款。
- 卖家同意后，订单变为 `refunded`，支付流水追加退款记录，商品恢复为 `on_sale`。
- 卖家拒绝后，订单回到申请退款前的状态。
- 买家可以撤回退款申请。
- 退款申请中禁止直接确认收货，避免资金状态冲突。

### 4. 交易申诉 / 客服仲裁

- 买家或卖家可以在订单详情提交申诉。
- 管理员在后台“交易申诉”中查看申诉原因、订单信息和双方身份。
- 管理员填写处理说明后，可将申诉处理为“已处理”或“不受理”。
- 处理结果会通知买卖双方，并写入管理员操作日志。

### 5. 评价流程

- 订单完成后，买卖双方可以互评。
- 同一订单中同一评价人只能评价一次。
- 商品详情页展示卖家近期评价和评分。

## 六、功能模块

### 1. 商品模块

- 发布商品
- 编辑商品
- 删除无订单商品
- 下架 / 重新上架自己的商品
- 管理员下架商品并填写原因
- 管理员恢复商品
- 卖家可查看管理员下架原因
- 管理员下架的商品，卖家不能自行恢复或删除
- 用户举报商品后会弹窗确认，管理员可在后台继续审核处理

### 2. 图片模块

- 支持本地上传商品图片
- 一个商品最多 4 张图片
- 编辑商品时可保留、删除、追加图片
- 图片路径保存在 `product_images` 表
- `products.image_url` 保留兼容字段
- 用户上传目录为 `static/uploads/`
- 演示商品图片目录为 `static/product_images/`

### 3. 收藏模块

- 用户可以收藏商品
- “我的收藏”可集中查看收藏商品
- 同一用户对同一商品只能收藏一次
- 收藏页取消收藏后仍停留在收藏页

### 4. 消息模块

- 商品详情页可以联系卖家，发送后自动进入与卖家的对话
- 订单详情中买家可以联系卖家，卖家可以联系买家
- “我的消息”按联系人分组展示，类似聊天会话
- 打开某个会话只会把该联系人消息标记为已读
- 删除对话只影响当前用户视角，不删除对方消息

### 5. 提醒中心

- 顶部有“提醒”入口
- 侧边栏有“提醒中心”入口
- 未读消息、系统事件、待发货订单、待处理退款会合并显示数量
- 点击提醒后跳转到对应订单、商品或消息会话

### 6. 个人中心

- 账户总览
- 交易订单
- 我的发布
- 收藏夹
- 我的消息
- 提醒中心
- 钱包与资料
- 修改昵称、登录密码；修改手机号和设置/修改支付密码进入独立安全验证页面
- 钱包充值

个人中心采用工作台布局，左侧导航提供“返回首页”入口，避免用户误以为个人中心就是整个网站。

### 7. 管理员后台

管理员后台入口只对 `admins` 表中的用户显示。

后台主要功能：

- 后台总览：展示用户、商品、待处理举报、交易申诉、平台态势和最近订单
- 商品管理：查看商品、搜索商品、下架商品、恢复商品
- 商品举报：查看用户举报，处理为成立或不成立；举报成立时可联动下架商品
- 订单管理：查看全站订单，按状态筛选，按订单号、商品、买家、卖家搜索
- 交易申诉：处理买卖双方订单争议，填写客服仲裁说明
- 用户管理：搜索用户、启用 / 停用用户、设置管理员身份和权限
- 系统设置：控制是否开放管理员注册入口
- 操作日志：展示管理员关键操作，包括谁下架、谁恢复、谁改权限、谁处理举报和申诉

管理员权限字段：

| 字段 | 说明 |
|---|---|
| `can_manage_products` | 商品管理、商品举报处理权限 |
| `can_manage_users` | 用户管理、权限配置权限 |
| `can_manage_admin_register` | 管理员注册入口控制权限 |

## 七、数据库表

当前版本共 15 张核心表：

| 表名 | 说明 |
|---|---|
| `users` | 用户账号、登录密码哈希、支付密码哈希（可为空，首次支付前设置）、昵称、手机号、余额、启停状态 |
| `admins` | 管理员身份和后台权限 |
| `categories` | 商品分类 |
| `products` | 商品主表 |
| `product_images` | 商品多图 |
| `favorites` | 收藏关系 |
| `orders` | 订单 |
| `payments` | 支付与退款流水 |
| `app_settings` | 系统设置 |
| `reviews` | 评价 |
| `messages` | 站内消息 |
| `notifications` | 事件提醒 |
| `admin_logs` | 管理员操作日志 |
| `product_reports` | 商品举报 |
| `order_appeals` | 交易申诉 |

重点关系：

- `users` 与 `admins` 是 1 对 0/1 关系
- `products` 与 `product_images` 是 1 对多关系
- `users` 与 `favorites`、`messages`、`reviews` 存在多种业务关系
- `orders` 关联买家、卖家、商品、支付流水和评价
- `product_reports` 关联举报人、商品、处理管理员
- `order_appeals` 关联订单、申诉人、处理管理员
- `admin_logs` 记录管理员对商品、用户、举报、申诉和系统设置的关键操作

## 十、常见问题

| 现象 | 原因 | 解决方式 |
|---|---|---|
| 连接不上 MySQL | MySQL 服务未启动 | 启动 MySQL 服务或使用一键启动脚本 |
| `Unknown database secondhand` | 数据库未初始化 | 执行 `python init_db.py` |
| `Access denied` | 数据库账号或密码不正确 | 检查 `config.py` |
| 5000 端口被占用 | 已有 Flask 进程运行 | 关闭旧进程或重启一键启动脚本 |
| 图片不显示 | 上传文件不在 static 目录或路径错误 | 使用发布/编辑页面重新上传图片 |
| 管理员无法进入后台 | 当前用户不在 `admins` 表中 | 使用 `admin/admin` 或在用户管理中授予管理员身份 |
