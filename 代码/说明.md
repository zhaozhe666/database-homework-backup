# 校园二手交易平台（数据库课程设计）

基于 **Flask + MySQL 5.6** 的校园二手交易系统。当前版本已经从最初的商品交易闭环扩展为：商品多图、收藏、站内消息、提醒中心、10 分钟支付倒计时、退款、评价、个人资料修改、管理员后台和独立管理员权限表。

## 一、技术栈

- 后端：Python + Flask
- 数据库：MySQL 5.6，InnoDB，支持事务与外键
- 数据库驱动：PyMySQL
- 前端：Jinja2 模板 + 原生 CSS
- 图片：用户上传保存到 `static/uploads/`，初始化演示商品图保存在 `static/product_images/`
- 表单安全：所有会修改数据的 POST 表单都带 CSRF token 校验

### 0. 一键启动

在 `代码` 文件夹里双击：

```text

注意：一键启动不会自动执行 `init_db.py`，因为初始化脚本会清空并重建数据库。首次运行或需要重置演示数据时，才手动执行初始化。

cd /d "C:\Users\赵喆\Desktop\数据库作业\代码"

pip install -r requirements.txt
```

```cmd

```

```cmd

```

## 五、核心业务流程

### 商品交易流程

1. 卖家发布商品，商品状态为 `on_sale`。
2. 买家下单，商品状态变为 `locked`，订单状态为 `created`。
3. 买家需要在 10 分钟内支付。
4. 支付成功后，买家余额扣款，订单状态变为 `paid`。
5. 买家确认收货后，金额打给卖家，商品变为 `sold`，订单变为 `completed`。

### 超时取消

- 待支付订单超过 10 分钟未付款，会自动取消。
- 商品从 `locked` 恢复为 `on_sale`。
- 订单状态变为 `cancelled`。
- 订单详情页有实时倒计时。

### 退款流程

- 买家已付款但未确认收货时，可以申请退款。
- 订单状态变为 `refund_requested`。
- 卖家在提醒中心和订单详情中看到退款申请。
- 卖家同意退款后，金额退回买家余额，商品恢复 `on_sale`，订单变为 `refunded`。
- 卖家也可以拒绝退款，订单回到 `paid` 待收货状态。
- 买家可以撤回退款申请，或在退款申请中直接确认收货，避免订单卡在 `refund_requested`。

### 评价流程

- 订单完成后，买卖双方可以互评。
- 同一订单同一评价人只能评价一次。
- 商品详情页显示卖家近期评价和评分。

## 六、功能模块

### 商品模块

- 发布商品
- 编辑商品
- 删除无订单商品
- 下架商品
- 重新上架自己下架的商品
- 管理员下架商品并填写原因
- 卖家可以看到管理员下架原因
- 管理员下架的商品，卖家不能自行重新上架或删除

### 图片模块

- 支持本地上传图片
- 一个商品最多 4 张图片
- 可以分多次添加图片
- 编辑商品时可以保留、删除、追加图片
- 图片路径保存到 `product_images` 表
- `products.image_url` 仅保留兼容缓存
- 用户上传图片目录是 `static/uploads/`，初始化演示商品图目录是 `static/product_images/`
- 上传文件保存时使用系统生成的新文件名，中文原文件名只用于判断扩展名，避免合法的 `图片.jpg` 被误判格式。

### 收藏模块

- 买家可以收藏商品
- 个人中心可以进入“我的收藏”
- 同一用户对同一商品只能收藏一次

### 消息模块

- 商品详情页可以联系卖家
- 我的消息按联系人分组，类似 QQ 对话框
- 点开某个联系人，只会把该联系人对话设为已读
- 其他未读消息会继续保留红色提醒
- 对话可以删除，删除只影响当前用户视角，不删除对方消息

### 提醒模块

- 顶部导航有“提醒”入口
- 未读消息和待处理退款会合并显示红色数量
- 提醒中心中：
  - 消息提醒跳转到对应联系人对话
  - 退款提醒跳转到对应订单详情

### 个人中心

- 查看在售商品、买到的、卖出的、收藏、信息
- 信息卡片右上角显示未读数量
- 修改昵称、手机号
- 修改密码
- 钱包充值

### 管理员后台

管理员后台入口只对 `admins` 表中的用户显示。

后台功能：

- 商品管理：查看商品、搜索商品、下架商品、填写下架原因、恢复商品
- 订单管理：查看全站订单，按状态筛选，并按订单号、商品、买家、卖家搜索
- 用户管理：搜索用户、启用/停用用户，把普通用户设为管理员，配置管理员权限
- 系统设置：控制是否开放管理员注册入口

管理员权限字段：

- `can_manage_products`：商品管理权限
- `can_manage_users`：用户管理权限
- `can_manage_admin_register`：管理员注册控制权限

## 十、常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| 连接不上 MySQL | MySQL 服务未启动 | 启动 MySQL 服务 |
| `Unknown database secondhand` | 未初始化数据库 | 执行 `python init_db.py` |
| `Access denied` | 数据库账号或密码不对 | 检查 `config.py` |
| 5000 端口被占用 | 已有 Flask 进程在运行 | 关闭旧进程或旧命令行窗口 |
| 图片不显示 | 上传文件不在 static 下 | 使用发布/编辑页面本地上传 |
